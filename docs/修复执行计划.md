### P0-2. 锁定唯一 Agent 配置

**关联问题：** B2、H7、A2（多套配置漂移）
**预计时间：** 4-6 小时
**验收条件：** 只有一个 Agent 工厂函数；配置源是官方 YAML 快照 + 显式锁定的 SHA；不允许手写模板

#### 关键设计决定

- ❌ 不自动跟随官方 YAML 更新（科研实验要求配置锁定）
- ❌ 工厂不返回 `ConDiagIntegratedAgent`（旧 v4 冻结模块）
- ✅ 返回 `minisweagent.agents.default.DefaultAgent` 或自定义薄包装
- ✅ 工厂返回 `Callable[[], DefaultAgent]` 不是 agent 实例（每个分支需要独立创建）
- ✅ 必须区分 "复现旧 baseline" 和 "新 persistent revision" 两套协议参数

#### AgentConfig

```python
@dataclass(frozen=True)
class AgentConfig:
    protocol_name: str          # "baseline_reproduction" | "persistent_revision"
    protocol_version: str       # "1.0"
    model_name: str = "deepseek/deepseek-v4-pro"
```

### P0-3. Patch Provenance（四分 Patch 对象 + Agent 提交收集）

**关联问题：** B4（提交 vs 评测不一致）
**预计时间：** 4-5 小时
**验收条件：** 每个 episode 保存四份 Patch 及其 SHA，互相可追溯

#### 四个 Patch 对象

| 对象 | 来源 | 用途 |
|------|------|------|
| `agent_submitted_patch` | Agent 通过 `cat patch.txt` 提交的内容 | Agent 自认为提交了什么 |
| `evaluation_patch` | 经过完整性检查后送入 Harness 的 Patch | 官方评测实际跑了什么 |
| `workspace_snapshot` | Round 1 完成时容器完整状态（tracked + untracked） | Round 2 应该恢复什么 |
| `final_evaluation_patch` | Round 2 完成时经检查后送入 Harness 的 Patch | 最终评测 |

#### Agent 提交收集

单一 collector 从多个候选来源提取：

```python
def collect_agent_submission(agent, submitted_exception, messages) -> AgentSubmission:
    """从所有可能来源收集 agent 提交的 patch，选择最终使用的版本。"""

class AgentSubmission:
    exception_payload: str        # Submitted 异常中的 payload
    patch_file_content: str       # 容器内 /testbed/patch.txt
    submit_command_output: str    # 最后一个提交命令的输出
    selected_patch: str           # 最终选用的版本
    selected_source: str          # 选用的来源名称
    consistency_status: str       # consistent / SUSPICIOUS_SUBMISSION_MISMATCH
```

候选内容不一致时标记 `SUSPICIOUS_SUBMISSION_MISMATCH`，不能默默选一个。

#### Evaluation Patch 的生成原则

- ❌ 不允许自动"过滤"agent 提交的 patch（静默删除 reproduction script 等文件再送评测）
- ✅ 合法 submission → `evaluation_patch = canonicalize(agent_submitted_patch)`（仅换行/排序等格式化）
- ✅ 非法 submission（test-only / artifact-only / empty）→ 阻断实验
- ✅ 要求 `semantic_patch_sha(agent_submitted) == semantic_patch_sha(evaluation_patch)`

### P0-5. Workspace Snapshot（含 tracked + untracked）

**关联问题：** B5（restore 内容不完整）
**预计时间：** 4-5 小时
**验收条件：** Snapshot 同时包含 tracked diff 和 untracked 文件内容归档；SHA 覆盖全部状态

```python
@dataclass
class UntrackedFile:
    path: str
    size: int
    sha256: str

@dataclass
class WorkspaceSnapshot:
    tracked_diff: str                            # git diff --binary --no-ext-diff
    untracked_manifest: list[UntrackedFile]       # 未跟踪文件清单
    untracked_archive_path: str | None            # 未跟踪文件 tar 包路径
    base_commit_sha: str                          # git rev-parse HEAD

    # SHA 同时覆盖 tracked + untracked
    @property
    def tracked_diff_sha(self) -> str:
        return _sha(self.tracked_diff)

    @property
    def untracked_manifest_sha(self) -> str:
        return _sha(json.dumps([u.sha256 for u in self.untracked_manifest]))

    @property
    def workspace_state_sha(self) -> str:
        return _sha(self.base_commit_sha + self.tracked_diff_sha + self.untracked_manifest_sha)
```

保存 snapshot 时，将允许保留的 untracked 文件打成 tar 存档。

排除规则：

```
patch.txt
__pycache__/
*.pyc
*.pyo
构建产物（.so, .o, .egg-info/）
单文件 > 1MB
.git/
venv/ .venv/
node_modules/
```

#### 多指纹验证

Fairness Gate 比较 `workspace_state_sha`，不只比较 `tracked_diff_sha`。

分别保存：

```
tracked_diff_sha
untracked_manifest_sha
workspace_state_sha
```

### P0-6. Workspace Restore + Preflight Fairness（run_branch 内部）

**关联问题：** B5/B6/B7
**预计时间：** 6-8 小时
**验收条件：** Restore 成功后验证 `workspace_state_sha`；Fairness 在第一次 LLM 调用前完成

#### Restore 实现（NamedTemporaryFile + docker cp + untracked tar）

```python
def restore_workspace(agent, snapshot: WorkspaceSnapshot, base_commit: str) -> RestoreResult:
    temp_path = None
    try:
        # 保护 1: 验证容器 HEAD
        head = agent.env.execute({"command": "cd /testbed && git rev-parse HEAD"})
        if head["output"].strip() != base_commit:
            return RestoreResult(ok=False, reason="HEAD != base_commit")

        # 保护 2: 清理工作区
        agent.env.execute({"command": f"cd /testbed && git reset --hard {base_commit}"})
        agent.env.execute({"command": "cd /testbed && git clean -fd"})

        # 恢复 tracked diff
        with tempfile.NamedTemporaryFile(mode="w", suffix=".patch", delete=False) as f:
            f.write(snapshot.tracked_diff)
            temp_path = f.name
        r1 = subprocess.run(["docker", "cp", temp_path, f"{cid}:/tmp/restore.patch"], ...)
        if r1.returncode != 0:
            return RestoreResult(ok=False, reason="docker cp failed")
        r2 = agent.env.execute({"command": "cd /testbed && git apply --check /tmp/restore.patch && git apply --whitespace=nowarn /tmp/restore.patch"})
        if r2["returncode"] != 0:
            return RestoreResult(ok=False, reason=f"git apply failed: {r2['output']}")

        # 恢复 untracked 文件
        if snapshot.untracked_archive_path and os.path.exists(snapshot.untracked_archive_path):
            subprocess.run(["docker", "cp", snapshot.untracked_archive_path, f"{cid}:/tmp/untracked.tar"], ...)
            agent.env.execute({"command": "cd /testbed && tar xf /tmp/untracked.tar 2>/dev/null || true"})

        # 验证 workspace_state_sha
        r = agent.env.execute({"command": f"cd /testbed && git diff --binary --no-ext-diff {base_commit}"})
        if r["returncode"] != 0:
            return RestoreResult(ok=False, reason="git diff failed after restore")
        restored_sha = _sha(base_commit + r["output"] + json.dumps(snapshot.untracked_manifest))
        if restored_sha != snapshot.workspace_state_sha:
            return RestoreResult(ok=False, reason=f"workspace_state_sha mismatch")

        return RestoreResult(ok=True, workspace_sha=restored_sha)
    finally:
        if temp_path and os.path.exists(temp_path):
            os.unlink(temp_path)

@dataclass
class RestoreResult:
    ok: bool
    workspace_sha: str = ""
    reason: str = ""
```
