### P0-4. Workspace Restore 重写

**关联问题：** B5
**预计时间：** 6-8 小时（最复杂的单项）
**验收条件：** 分支恢复后容器内 `git diff` SHA 与 R1 一致

**修复方案选型：**

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| A. NamedTemporaryFile → `docker cp` | 接口标准、可靠 | 需宿主临时文件 | ✅ 推荐 |
| B. Base64 → `echo | base64 -d` | 纯容器内操作 | 大 patch 效率低 | 备选 |
| C. tar archive → `docker cp -` | 官方支持 | tar 构造复杂 | 不推荐 |

**方案 A 实现：**

```python
def _apply_patch(agent, patch_text: str, base_commit: str) -> bool:
    """恢复 R1 workspace 并验证 SHA。返回 True/False。"""
    with tempfile.NamedTemporaryFile(mode="w", suffix=".diff", delete=False) as f:
        f.write(patch_text)
        f.flush()
        # Step 1: 复制 diff 到容器
        r1 = subprocess.run(
            ["docker", "cp", f.name, f"{container_id}:/tmp/restore.diff"],
            capture_output=True, timeout=10,
        )
        if r1.returncode != 0:
            logger.error("docker cp failed: %s", r1.stderr)
            return False

        # Step 2: apply --check 验证
        r2 = agent.env.execute({"command": "cd /testbed && git apply --check /tmp/restore.diff 2>&1"})
        if r2["returncode"] != 0:
            logger.error("git apply --check failed: %s", r2["output"])
            return False

        # Step 3: apply
        r3 = agent.env.execute({"command": "cd /testbed && git apply --whitespace=nowarn /tmp/restore.diff 2>&1"})
        if r3["returncode"] != 0:
            return False

        # Step 4: 验证恢复后 SHA
        r4 = agent.env.execute({"command": f"cd /testbed && git diff --binary {base_commit} 2>/dev/null"})
        restored_sha = _sha(r4.get("output", ""))
        expected_sha = _sha(patch_text)
        if restored_sha != expected_sha:
            logger.error("Workspace restore SHA mismatch: %s != %s", restored_sha, expected_sha)
            return False

        return True
```

**错误策略：** 恢复失败 → `reason = "workspace_restore_failed"` → 终止实验，不进评测。

### P0-5. 真实 Fairness Gate

**关联问题：** B6、B7
**预计时间：** 2-3 小时
**验收条件：** 两个分支第一次 LLM 调用前的 workspace SHA 被捕获并比较

执行步骤：

1. 分支创建容器后、第一次 `agent.step()` 前，提取容器内 `git diff` SHA：

```python
def capture_workspace_sha(agent, base_commit: str) -> str:
    r = agent.env.execute({"command": f"cd /testbed && git diff --binary {base_commit} 2>/dev/null | sha256sum"})
    return r.get("output", "").split()[0]
```

2. `experiment.py` 中：
```python
sf_sha = capture_workspace_sha(sf_agent, base_commit)
cd_sha = capture_workspace_sha(cd_agent, base_commit)
# 先跟 R1 SHA 比
assert sf_sha == r1_sha, f"SF workspace mismatch: {sf_sha} != {r1_sha}"
assert cd_sha == r1_sha, f"CD workspace mismatch: {cd_sha} != {r1_sha}"
# 再互相比
assert sf_sha == cd_sha, f"SF/CD workspace differ"
```

3. `episode_manifest.json` 中记录三份 SHA：`r1_workspace_sha`、`sf_workspace_sha`、`cd_workspace_sha`
